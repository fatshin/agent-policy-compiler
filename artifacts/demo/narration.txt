Agent Policy Compiler turns operating rules into decisions an agent can execute and a reviewer can audit. This fixture contains five policy clauses and seven runtime scenarios. When I run the analysis, the compiler produces three allowed actions, two blocked actions, and two actions that require human approval. Every restriction cites the exact policy sentence that caused it. If a required policy domain is missing, compilation stops instead of guessing. I used Codex with GPT-5.6 to define the policy representation, implement the compiler and evaluator, generate boundary tests, and review the public evidence. A live GPT-5.6 extension can draft the representation, while deterministic validation remains authoritative.
